## Installation

<Tabs>
  <TabItem label="Go">
    Requires Go 1.24+.

    ```bash
    go install ancla.dev/cli@latest
    ```
  </TabItem>
  <TabItem label="Python">
    ```bash
    uv tool install ancla
    ```

    Or with pip:

    ```bash
    pip install ancla
    ```
  </TabItem>
  <TabItem label="Homebrew">
    ```bash
    brew install SideQuest-Group/ancla-client/ancla
    ```
  </TabItem>
  <TabItem label="npm / Bun">
    Run without installing:

    ```bash
    npx ancla
    # or
    bunx ancla
    ```

    Or install globally:

    ```bash
    npm install -g ancla
    # or
    bun install -g ancla
    ```
  </TabItem>
  <TabItem label="Cargo">
    Requires [cargo-binstall](https://github.com/cargo-bins/cargo-binstall).

    ```bash
    cargo binstall ancla
    ```
  </TabItem>
  <TabItem label="Binary">
    Download from [GitHub Releases](https://github.com/SideQuest-Group/ancla-client/releases):

    <Tabs>
      <TabItem label="macOS (Apple Silicon)">
        ```bash
        curl -Lo ancla.tar.gz https://github.com/SideQuest-Group/ancla-client/releases/latest/download/ancla_darwin_arm64.tar.gz
        tar xzf ancla.tar.gz

        sudo mv ancla /usr/local/bin/
        ```
      </TabItem>
      <TabItem label="macOS (Intel)">
        ```bash
        curl -Lo ancla.tar.gz https://github.com/SideQuest-Group/ancla-client/releases/latest/download/ancla_darwin_amd64.tar.gz
        tar xzf ancla.tar.gz
        chmod +x ancla
        sudo mv ancla /usr/local/bin/
        ```
      </TabItem>
      <TabItem label="Linux (amd64)">
        ```bash

```

## Quick start

### 1. Log in

```bash
ancla login
```

This opens your browser for authentication. See the [Authentication guide](/guides/authentication/) for CI and manual options.

### 2. List your workspaces

```bash
ancla workspaces list
```

### 3. List projects

```bash
ancla projects list --workspace my-ws
```

### 4. Deploy a service

```bash
ancla services deploy my-ws/my-project/production/my-service
```

### 5. Check deploy status

```bash
ancla services status my-ws/my-project/production/my-service
```

### 6. Link your project directory

```bash
ancla link my-ws/my-project/production/my-service
```

Now commands like `status`, `logs`, `run`, and `down` know what service you're working on without extra arguments. See [Project Linking](/guides/project-linking/) for details.
